## Description

Adds a PATCH `/widgets/:id` endpoint to the existing widget API on `Bontyyy/orchestrator-api-sample`. Clients can partially update a widget by sending a JSON body with any subset of the `{name, sku, quantity}` fields. Only provided fields are updated; omitted fields retain their current values. The endpoint reuses the existing `Widget` model and `InMemoryWidgetRepository` and enforces the same field-level validation rules as the create endpoint.

## Scope

- A single new endpoint `PATCH /widgets/:id` on the existing `WidgetsController`.
- The request body accepts any non-empty subset of `{name, sku, quantity}`. Omitted fields are left unchanged.
- On success, the endpoint returns HTTP 200 with the full updated widget JSON.
- On a non-existent `:id`, the endpoint returns HTTP 404 with `error.code = "widget_not_found"`.
- Field-level validation reuses the existing `CreateWidget` rules: `name` non-blank, `sku` non-blank, `quantity` in the range 0–10000.
- Uses the existing `Widget` model and `InMemoryWidgetRepository` without modification.

## Out of scope

- Full replacement semantics (PUT).
- Bulk partial updates.
- ETag or concurrency-control headers.
- Field-level validation rules beyond those already enforced by `CreateWidget`.
