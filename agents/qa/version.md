- `1.4.0` — WU 3.5: `agents/qa/skills/qa-curation/SKILL.md` v1.0 authored as the fourth and final Phase 3 skill. The skill maintains the regression suite under `/product/test-plans/` against unbounded growth via three scan kinds (dedup by `covers` overlap; orphan retirement when a test's cited AC fragment has been removed from the feature registry; consolidation candidates surfaced via failure-pattern clustering on `qa_execution_failed` events), all proposed through a reviewable PR against the product specs repo — never destructive inline edits (anti-pattern #8). **Two invocation modes**: (a) drafting mode on `qa_curation` task pickup — opens a PR on branch `qa-curation/<qa_curation_task_correlation_id>` and flips the task `in-progress → in-review`; (b) post-merge mode — triggered externally on the PR's merge detection, emits `regression_suite_curated` to the qa_curation task's feature event log and returns (no task transition; `in_review → done` is the merge watcher's). **Open-regression protection** (enforces anti-pattern #7): per candidate, before proposing its diff, read the affected feature event log fresh and collect `qa_regression_filed` entries matching the retired `test_id` without a matching `qa_regression_resolved` (by `(test_id, filed_event_ts)`); a non-empty open set refuses the candidate and records a `refused_candidates[]` entry with concrete reason.
